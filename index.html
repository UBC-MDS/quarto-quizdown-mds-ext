quizdown for Quarto — MDS Extension

Interactive quizzes for Quarto documents, powered by quizdown-js. Fork of parmsam/quarto-quizdown, extended with new question types and a print review mode.

Install

quarto add UBC-MDS/quarto-quizdown-mds-ext

Then add the filter to your document front matter:

---
filters:
  - quizdown
---

Update an existing installation:

quarto update extension UBC-MDS/quarto-quizdown-mds-ext

Question types

All questions live inside a ```quizdown fenced code block. A single block can contain multiple questions — students navigate with the arrow buttons and submit all at once.

Single choice

Use a numbered list where exactly one item is marked [x]. Optionally add a > blockquote under any answer as per-option feedback.

NoteShow syntax 
```quizdown
# What is 2 + 2?

1. [ ] 3
1. [x] 4
    > Correct — basic arithmetic!
1. [ ] 5
```

Multiple choice

Use an unordered list (with -) where one or more items are marked [x].

NoteShow syntax 
```quizdown
## Which are Python mutable types?

- [x] list
    > Lists can be modified after creation.
- [ ] tuple
    > Tuples are immutable.
- [x] dict
    > Dictionaries are mutable.
- [ ] str
    > Strings are immutable in Python.
```

Matching (1-to-1)

Use an unordered list with Left :: Right pairs. Each right chip belongs to exactly one left prompt. Add a distractor with :: Value (no left side). Per-pair feedback uses an indented > blockquote.

NoteShow syntax 
```quizdown
## Match each term to its definition!

- Term A :: Definition A
    > Feedback for this pair.
- Term B :: Definition B
- :: Red herring
    > Feedback shown when this distractor is placed.

> Overall question hint.
```
--- shuffle_answers: true --- ## Match each Python data structure to its key property! - list :: Ordered, mutable sequence > Lists maintain insertion order and can be changed after creation. - tuple :: Ordered, immutable sequence > Tuples are like lists but cannot be modified once created. - dict :: Key-value store with unique keys > Dictionaries map unique keys to values; order is preserved in Python 3.7+. - set :: Unordered collection of unique values > Sets automatically deduplicate — great for membership tests. - :: Linked list > Python has no built-in linked list (though `collections.deque` is close). > Think about mutability and ordering.

Matching (multi-match / classification) (MDS addition)

When the same right-side label appears for multiple left items, it automatically becomes a reusable chip that stays in the pool after being placed — perfect for classify-into-bins tasks.

NoteShow syntax 
```quizdown
## Classify each task!

- Task A :: Category 1
    > Feedback for Task A.
- Task B :: Category 2
    > Feedback for Task B.
- Task C :: Category 1
    > Feedback for Task C — same chip as Task A.
- :: Red herring
    > This option is never correct.
```
--- shuffle_answers: true --- ## Classify each ML task as Regression or Classification! > Regression predicts a **continuous** value; Classification predicts a **category**. - Predict house price :: Regression > The target (price) is a continuous number. - Detect spam email :: Classification > The output is a category: spam or not spam. - Forecast stock return :: Regression > Returns are continuous values, not categories. - Identify handwritten digit :: Classification > Digits 0–9 are discrete categories. - Estimate delivery time :: Regression > Delivery time is a continuous quantity. - Diagnose disease from image :: Classification > Disease/no-disease is a categorical outcome. - :: Clustering > Clustering is unsupervised — there is no target label to predict.
